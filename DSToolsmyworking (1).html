Explanation of the above so long error right now

The error occurs because you are trying to compute the mean of a DataFrame that includes non-numeric columns. Specifically, you are using:

df[['age']] = df[['age']].fillna(df.mean())  # Fill NaNs with mean

Issues:

  1. df.mean() calculates the mean for all numeric columns in the DataFrame. However, df contains non-numeric columns, like categorical variables (e.g., occupation, gender), which cause pandas to throw a TypeError.
  2. You are applying df.mean() to a specific column (age), but since df.mean() returns a Series of means for all numeric columns, pandas might not be able to match the right value for age.

Solution:

Since you only want to fill missing values in the age column, modify your code as follows:

df['age'] = df['age'].fillna(df['age'].mean())  # Fill NaNs in 'age' only
df['age'] = df['age'].ffill()  # Forward fill remaining NaNs if any

Explanation:

  • df['age'].mean() ensures that only the mean of the age column is calculated.
  • fillna(df['age'].mean()) replaces NaN values in age with the computed mean.
  • ffill() ensures any remaining NaN values are filled using forward fill.

This fixed the issue.
